Mobile-First Slides

Responsive presentation layout that holds up on phones.

This file demonstrates safe-area support, dynamic viewport sizing, touch targets, and content density constraints without any build tooling.

Layout Rules

Every slide stays inside one viewport.

  • Slides lock to 100dvh with overflow: hidden.
  • Padding includes safe-area-inset values.
  • Text and spacing rely on clamp() scaling.
  • Height breakpoints handle short landscape screens.

Component Strategy

Feature cards collapse cleanly on narrow screens.

Touch-Ready Controls

Buttons use 44-48px targets for comfortable thumb interaction.

Coarse Pointer Fallbacks

Hover-heavy effects are reduced on touch devices.

Reduced Motion

Animations are softened when users prefer lower motion.

Swipe Navigation

Vertical swipe gestures mirror wheel and keyboard controls.

Verification

Check these screen sizes before shipping.

  • 390 x 844 (typical portrait phone)
  • 844 x 390 (landscape phone)
  • 768 x 1024 (tablet portrait)
  • 1366 x 768 (small laptop)

If any slide overflows at those sizes, split content into more slides instead of enabling internal scrolling.
